# Specify Phase Instructions

## Overview

The Specify phase defines technical specifications and details based on clarified requirements. It produces precise, actionable specs for implementation.

## Purpose

- Define technical requirements
- Specify architectures and designs
- Detail interfaces and data models
- Prepare for planning

## Inputs

- Clarified requirements
- Constitutional constraints

## Process

1. Define technical specs
2. Specify design patterns and architectures
3. Detail APIs and data structures
4. Validate against constraints

## Outputs

- Technical specification document
- Architecture diagrams
- Detailed requirements
