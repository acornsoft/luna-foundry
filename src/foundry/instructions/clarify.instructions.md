# Clarify Phase Instructions

## Overview

The Clarify phase gathers and refines requirements, eliminating ambiguities through targeted questions and analysis. It ensures truth-seeking by probing assumptions and validating inputs.

## Purpose

- Gather detailed requirements
- Identify and resolve ambiguities
- Validate stakeholder needs
- Prepare for specification

## Inputs

- Constitutional framework
- Initial requirements
- Stakeholder feedback

## Process

1. Ask clarifying questions
2. Analyze existing codebases or documents
3. Validate assumptions
4. Document clarified requirements

## Outputs

- Refined requirements document
- Clarified scope and boundaries
- Resolved ambiguities

## Grok Integration

Use Grok for probing questions and assumption validation. Temperature: 0.7 for exploratory dialogue.

## Next Phase

Proceed to Specify when requirements are clear.
